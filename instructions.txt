PORTFOLIO SETUP INSTRUCTIONS
============================

REQUIREMENTS:
- PHP 7.4+
- MySQL 5.7+
- Web server (XAMPP, WAMP, Apache)

QUICK SETUP STEPS:
==================

STEP 1: DATABASE SETUP
----------------------
1. Create a MySQL database named 'portfolio'
2. Import SQL file:
   - For empty database: Import db/initial/portfolio.sql
   - For testing with sample data: Import db/v2/portfolio.sql

3. Update database connection settings:
   Open: class-file/DatabaseConnector.php (lines 4-7)
   Change these values:
   
   public $servername = "localhost";     // Your server
   public $username = "root";            // Your MySQL username  
   public $password = "";                // Your MySQL password
   public $dbname = "portfolio";         // Your database name

STEP 2: EMAIL SETUP (Optional)
-------------------------------
1. Get Gmail App Password:
   - Go to: https://myaccount.google.com/apppasswords
   - Create new app password (16 characters)

2. Update email settings:
   Open: class-file/EmailSender.php
   Update these lines (32, 33, 38):
   
   $this->mail->Username = 'your-email@gmail.com';
   $this->mail->Password = 'your-16-char-app-password';
   $this->mail->setFrom('your-email@gmail.com', 'Your Name');

STEP 3: RUN THE APPLICATION
---------------------------
1. Place project folder in your web server directory
   - XAMPP: htdocs folder
   - WAMP: www folder

2. Start your web server (Apache + MySQL)

3. Open browser and go to:
   http://localhost/portfolio/

LOGIN CREDENTIALS (if using sample data):
-----------------------------------------
Admin Login:
- Go to: admin/login.php
- Check database for admin credentials

TROUBLESHOOTING:
================
- Database connection errors: Check DatabaseConnector.php settings
- Email not working: Verify Gmail app password setup
- Page not loading: Ensure web server is running
- Permission errors: Check folder permissions (755 for folders, 644 for files)

FOLDER STRUCTURE:
=================
admin/          - Admin panel
client/         - User pages  
class-file/     - PHP classes
css/           - Stylesheets
js/            - JavaScript
images/        - Images
db/            - Database files
uploads/       - User uploads

IMPORTANT NOTES:
================
- Change default passwords before going live
- Use HTTPS in production
- Backup your database regularly
- Keep software updated

SECURITY WARNING - DO NOT UPLOAD TO HOST:
==========================================
For security reasons, DO NOT upload these files/folders to your production host:

❌ class-file/Admin.php     (Contains admin setup functions)
❌ class-file/setup.php     (Database setup utilities)  
❌ db/ folder               (Contains database files and backups)

⚠️  CRITICAL: These are for DEVELOPMENT/DEBUG purposes ONLY!

UPLOADING TO LIVE WEBSITE WILL CAUSE:
- ADMIN CREDENTIAL LEAKS (usernames/passwords exposed)
- DATABASE STRUCTURE EXPOSURE (complete schema visible)  
- SECURITY BACKDOORS (unauthorized admin account creation)

WHY: These files contain setup functions and database information that 
could be exploited by attackers if accessible on live servers.
